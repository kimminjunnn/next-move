# Rupa UI Guide

## 톤

- 실제 실내 볼더링장에 들어온 듯한 친근한 느낌을 유지한다.
- Rupa는 귀여운 원숭이 클라이밍 파트너처럼 느껴져야 한다.
- 합판 벽, 피스 구멍, 컬러 홀드, 검은 볼륨의 현장감을 살린다.
- 과한 SaaS 랜딩 페이지 느낌을 피한다.
- 기능 앱의 명확함은 유지하되, 차갑거나 의료/분석 도구처럼 보이지 않게 한다.

## 색상 방향

- 주요 배경은 합판 클라이밍 벽 베이지다.
- 큰 빈 배경에는 피스 구멍이나 T-nut 점 질감을 촘촘하고 분명하게 사용한다.
- primary color는 노란 홀드/바나나 계열이다.
- primary 위 텍스트는 흰색보다 짙은 원숭이 브라운을 우선한다.
- 파란 홀드/초크백 색은 route/tool 보조 accent로 사용한다.
- 핑크/초록 홀드는 장식 또는 상태 강조에 제한적으로 쓴다.
- 빨강은 primary로 쓰지 않고 삭제/에러 같은 danger에만 사용한다.
- 검은색/짙은 회색은 볼륨, 매트, 사진 캔버스, 오버레이에 주로 사용한다.

## 문구

- 한국어 우선.
- 짧고 직접적으로 쓴다.
- 과장된 마케팅 문구를 피한다.
- “직접 시험해본다”는 느낌을 유지한다.
- 브랜드명은 `Rupa`로 통일한다.

## 홈 화면

- 홈은 진입용 랜딩이다.
- 핵심 메시지와 시작하기 행동을 중심에 둔다.
- 기능을 중복 배치하지 않는다.
- 홈 배경은 Rupa 분위기를 가장 강하게 보여주는 화면이다.
- `<HomeHeroHolds />`는 추상 장식보다 실제 클라이밍장 벽처럼 보여야 한다.
- 구조와 문구를 크게 바꾸기보다 배경, 질감, 색, 장식으로 분위기를 만든다.

## 내부 화면

- 실제 첫 행동은 사진 입력이다.
- 주기능과 보조기능의 위계를 명확하게 둔다.
- 사진/시뮬레이션 캔버스는 가독성이 우선이므로 어두운 작업 영역을 사용할 수 있다.
- 캔버스 안 action accent는 노란 primary와 파란 route/tool accent를 우선한다.
- 입력/설정 화면은 합판벽 배경, 따뜻한 카드 표면, 노란 primary, 파란 accent 액션으로 통일한다.

## React Native UI 규칙

- 반복 사용 화면은 모바일에서 손이 닿기 쉬운 조작 밀도를 유지한다.
- 버튼 텍스트가 좁은 화면에서 넘치지 않게 한다.
- 시뮬레이션 화면은 사진과 오버레이가 우선이며 장식은 이를 방해하지 않는다.
- 카드 안에 카드를 중첩하지 않는다.
- hero급 타이포그래피는 홈 같은 실제 hero 화면에만 쓴다.
- 도구 화면은 마케팅 섹션이 아니라 작업 표면처럼 구성한다.
