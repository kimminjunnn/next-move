# Git Workflow

## Branch

- feature/<name>
- fix/<name>

## Commit Rules

- 기능 단위로 작게 커밋
- unrelated changes 섞지 않기
- 커밋 전 diff 확인

## Commit Style

- feat: ...
- fix: ...
- refactor: ...
- design: ...

## Push

- push는 명시적으로 요청받을 때만 진행
- 위험한 변경은 먼저 확인

## Collaboration

- 작업 후:
  1. diff 요약
  2. 커밋 단위 제안
  3. 요청 시 커밋
  4. 요청 시 푸시
