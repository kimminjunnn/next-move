# Git Workflow

## Branch

- feature/<name>
- fix/<name>

## Commit Rules

- 기능 단위로 작게 커밋
- unrelated changes 섞지 않기
- 커밋 전 diff 확인

## Commit Style

- 커밋 메시지는 `깃모지 한글 요약` 형식을 기본으로 한다.
- 깃모지는 최소 세트만 사용한다.

| 변경 유형 | 깃모지 | 예시 |
| --- | --- | --- |
| 기능 개선 | ✨ | `✨ 스틱맨 캐릭터 렌더러 추가` |
| 구조 개선, 코드 포맷 적용 | 🎨 | `🎨 시뮬레이션 오버레이 구조 정리` |
| 버그 수정 | 🐛 | `🐛 머리 드래그 거리 계산 수정` |
| 문서 추가/수정 | 📝 | `📝 캐릭터 리소스 보존 정책 문서화` |
| 코드 리팩토링 | ♻️ | `♻️ 포즈 머리 방향 계산 분리` |

- 위 범주로 애매하게 나뉘지 않는 변경은 새 깃모지를 만들기보다 가장 가까운 범주를 고른다.
- 커밋 본문이 필요하면 무엇을 바꿨는지보다 왜 나눴는지와 검증 결과를 짧게 적는다.

## Push

- push는 명시적으로 요청받을 때만 진행
- 위험한 변경은 먼저 확인

## Collaboration

- 작업 후:
  1. diff 요약
  2. 커밋 단위 제안
  3. 요청 시 커밋
  4. 요청 시 푸시
